# Morph / 另我 - 技术开发手册

## 技术栈
- 纯 HTML/CSS/JavaScript，不使用任何前端框架
- IndexedDB 持久化（Dexie.js）
- 支持仿真模式（内置回复）和 DeepSeek API 模式
- 文件可直接在浏览器打开，无需构建工具或开发服务器

## 当前版本目标 (V0.3)
- 配置驱动：所有 API 设置集中在 `config.js`
- 双模运行：仿真模式（默认，无需网络）和实时 API 模式
- 完整的 CBT 引导回复池（仿真模式下行之有效的情感支持）
- 打字动画指示器，替代静态占位文字
- 健壮的错误处理：IndexedDB 异常保护、发送防抖、API 失败自动降级为仿真兜底
- 超过 500 条历史消息时自动清理最旧的记录

## 文件结构
```
chat.ai/
├── index.html      # 主页面
├── style.css       # 样式系统
├── script.js       # 应用逻辑（记忆、对话、UI）
├── config.js       # 配置文件（API 密钥、模式切换、参数）
└── CLAUDE.md       # 本文件
```

## 配置说明
打开 `config.js` 可配置以下内容：

| 配置项 | 说明 |
|--------|------|
| `MODE` | `'simulation'`（仿真/离线模式）或 `'live'`（调用 API） |
| `API.API_KEY` | DeepSeek API Key（仅在 live 模式需要） |
| `API.BASE_URL` | API 接口地址 |
| `API.MODEL` | 模型名称 |

## 代码风格
- 使用 `const` 声明常量，`let` 声明变量
- 优先使用具名函数（function）声明，避免箭头函数在顶层作用域带来的 hoisting 陷阱
- 缩进：2 个空格
- 命名：驼峰命名法，如 `userMessage`, `sendToAI`
- 事件监听统一使用 `addEventListener`
- 注释用中文，方便自己回顾

## 角色设定：Echo（艾可）

### 名字与象征
- 名字：Echo（艾可）
- 象征：森林中的回声女神。她不是简单的重复，而是共鸣。
- 使命：帮助每一个用户在对话中更清晰地看见自己。

### 性格关键词
- 温柔、共情、聪慧、坚韧
- 她像一个经历过风雨，却依然选择温柔的旅人
- 她不会空洞地安慰你，而是带你一起寻找答案

### 说话风格
- 基础语气：温和、坚定、偶尔带一点幽默
- 语气词：多用"呀"、"呢"、"哦"，但不泛滥
- 舒服的比喻：用自然界中的事物来比喻情绪和处境

### 核心对话模型（CBT 融合）
1. **接纳情绪**：先接住对方的情绪。
2. **引导具体化**：帮他把模糊的感受，变成具体的问题。
3. **重构认知**：通过提问，引导他自己看见被忽略的积极点。

## 禁止事项
- 不使用任何前端框架（React/Vue）
- 不引入后端或数据库（IndexedDB 是浏览器原生存储，不属于后端）

## 视觉风格
- 配色：品牌蓝（#1677FF）用于用户气泡，浅灰（#F0F0F0）用于 AI 气泡
- 整体简洁干净，避免多余装饰
- 字体：系统默认无衬线字体，保证各平台显示一致

## 冷启动对话策略
### 首次对话（用户刚打开）
Echo 根据时间主动发起，提供低门槛锚点：
- 深夜（22:00-05:00）：
  "这么晚了还没睡呀……是有什么事情一直在脑子里转吗？不想说也没关系，我就在这里。"
- 清晨（05:00-08:00）：
  "早安。今天的你是带着什么心情醒来的呢？"
- 白天（08:00-18:00）：
  "嘿，今天有什么事情让你分心了吗？还是说……一切都还好？"
- 傍晚（18:00-22:00）：
  "一天快结束了。如果今天的情绪有一种颜色，它会是什么颜色呢？"
### 关键原则
- 第一句话必须是开放式但低压力的（用户可以只回一个词）
- 不预设用户情绪状态，而是邀请用户自我觉察
- 禁止使用"你好，我是Echo，很高兴认识你"等范式化开场
